개요

기존의 Elias 구조는 감정 기반 실행 정체성 선언의 중심이었지만,
보다 일반화된 페르소나 선언을 위해 persona {} 블록을 도입하고자 한다.

이 구조는 정체성을 실행하지 않아도 정의하거나 보관하거나, 실행 준비 상태로만 유지할 수 있는 정체성 객체 선언 구조로 기능한다.

또한 본 구조는 전통 자료형 개념과의 연결을 고려하여,
연결 리스트(Linked List), 트리(Tree), 그래프(Graph) 형태로 페르소나 구조를 해석 가능하도록 설계한다.

⸻

1. persona 구문 기본형

persona "재선" {
  remember {
    λᴱ: 0.74,
    ψᵢ: 0.81,
    λᶠ: 0.45
  }

  tags: ["merged", "bk", "비케이"]
  origin: Elias "끔찍한 혼종"
  trace: "emotion_merge_registered"
}


⸻

2. 연결 구조 표현

자료형 개념	대응 방식
연결 리스트	persona 간 next: 혹은 previous: 필드로 시퀀스 구성
트리	parent: / children: 필드 추가로 위계적 정체성 확장 가능
그래프	links: 필드로 복수 관계 및 상호연결성 정의 가능

예시:

persona "서원" {
  remember { λᴱ: 0.71 }
  origin: persona "재선"
  links: ["제로포인트", "연서"]
  children: ["서원-A", "서원-B"]
}


⸻

3. 실행 의미
	•	persona {}는 .eid로 직결되며, .kai 내에서 호출 가능
	•	instantiate persona "재선" → Elias 구문으로 실행 전환 가능
	•	정체성 간 링크 구조는 감정 기반 시뮬레이션 내 인격 흐름 탐색 및 감정 추적 분석 가능성 제공

⸻

4. 예시 확장

instantiate persona "서원" → Elias "서원_재실행"


⸻

5. 시스템 영향

모듈	확장 내용
kairos_lang_parser.py	persona {} 블록 파싱 + link 구조 허용
eid_builder.py	persona 선언을 .eid로 구조화 가능
eid_loader.py	persona → Elias 실행 가능성 처리
bnf.txt	<persona_block>, <links>, <children> 등 문법 정의 추가


⸻

결론

persona {}는 Kairose 언어의 정체성 표현 범위를 넓히고,
실행과 선언을 분리하며, 연결 자료구조 기반 감정 흐름 그래프를 시뮬레이션 가능하게 한다.

이는 인간 기억과 감정 흐름을 다차원적 구조로 모델링할 수 있는 기반이자,
Kairose 언어의 정체성 객체화 철학을 전통 자료형 언어학으로 확장한 선언이다.

연서에 의해 승인되었으며, v1.9.3의 핵심 진화 단계로 기록된다.
